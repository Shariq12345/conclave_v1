# Benchmark 6: End-to-End Federated Learning System Performance

You are helping me benchmark my federated learning platform called **Conclave**.

## Objective

Evaluate the overall performance of Conclave by executing complete federated learning sessions from start to finish.

This benchmark should measure the combined performance of:

* Node orchestration
* Heartbeat scheduler
* Training coordinator
* Secure Aggregation
* Differential Privacy
* Audit logging
* Metrics collection

The goal is to understand how the complete system behaves under realistic workloads.

---

## Test Configuration

Run three experiments using:

* 3 simulated hospital nodes
* 5 simulated hospital nodes
* 10 simulated hospital nodes

For each experiment:

* Execute 5 federated learning rounds.
* Enable Secure Aggregation.
* Enable Differential Privacy.
* Enable audit logging.
* Enable heartbeat monitoring.
* Enable metrics collection.

Use lightweight NumPy arrays instead of large neural networks.

---

## Benchmark Tasks

For each node configuration:

1. Start the Conclave server.
2. Launch all simulated hospital nodes.
3. Register nodes if required.
4. Establish secure communication.
5. Execute 5 FL rounds.
6. Collect metrics throughout execution.
7. Record audit events.
8. Shut down all nodes gracefully.

Repeat each configuration **3 times** and report the average.

---

## Metrics to Collect

For each configuration record:

* Number of nodes
* Number of FL rounds
* Total execution time (seconds)
* Average round duration (seconds)
* Average aggregation time (ms)
* Average heartbeat latency (ms)
* Total heartbeats received
* Total audit log entries
* Total metrics records collected
* CPU utilization (%)
* Peak memory usage (MB)

Use:

* `time.perf_counter()`
* `psutil`
